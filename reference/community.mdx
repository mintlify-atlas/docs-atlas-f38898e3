## Contribute to Terra

Terra thrives on community contributions. There are many ways to get involved, regardless of your technical expertise.

### Ways to Contribute

<AccordionGroup>
  <Accordion title="Submit Package Requests">
    Want to see a package in Terra that isn't currently available?

    1. Check if the package has already been requested in [GitHub Issues](https://github.com/terrapkg/packages/issues)
    2. Search existing packages to ensure it's not already available
    3. Submit a new package request with details about the software
    4. Discuss your request with the community

    No packaging experience needed!
  </Accordion>

  <Accordion title="Report Bugs and Issues">
    Help improve Terra by reporting bugs you encounter:

    - Use the [GitHub Issues](https://github.com/terrapkg/packages/issues) page
    - Provide detailed information about your system and the problem
    - Include error messages and logs when relevant
    - Be responsive to questions from maintainers

    Quality bug reports help maintainers fix issues faster.
  </Accordion>

  <Accordion title="Contribute Package Manifests">
    If you have packaging experience or want to learn:

    1. Read the [Contribution Guide](https://developer.fyralabs.com/terra/contributing)
    2. Review the [Guidelines](https://developer.fyralabs.com/terra/guidelines)
    3. Fork the repository and create package manifests
    4. Submit a pull request with your changes

    The community is happy to help new packagers learn!
  </Accordion>

  <Accordion title="Improve Documentation">
    Help make Terra easier to use by improving documentation:

    - Fix typos and clarify unclear sections
    - Add examples and use cases
    - Write tutorials and guides
    - Translate documentation to other languages

    Good documentation makes Terra accessible to everyone.
  </Accordion>

  <Accordion title="Help Other Users">
    Share your knowledge and experience:

    - Answer questions in chat channels
    - Share solutions to problems you've solved
    - Write blog posts or tutorials about using Terra
    - Contribute to community discussions

    Your experience can help newcomers get started with Terra.
  </Accordion>
</AccordionGroup>

## Code of Conduct

The Terra community is committed to being inclusive and welcoming to all contributors.

### Our Expectations

<AccordionGroup>
  <Accordion title="Expected Behavior">
    Community members are expected to:

    - Participate in an authentic and active way
    - Exercise consideration and respect in speech and actions
    - Attempt collaboration before conflict
    - Refrain from demeaning, discriminatory, or harassing behavior
    - Be mindful of fellow participants and community guidelines

    These behaviors contribute to a healthy and positive community.
  </Accordion>

  <Accordion title="Unacceptable Behavior">
    The following behaviors are not tolerated:

    - Violence, threats, or violent language
    - Discriminatory jokes and language
    - Posting inappropriate content
    - Personal insults or attacks
    - Harassment of any kind
    - Sustained disruption of community events

    Unacceptable behavior will result in moderation action.
  </Accordion>

  <Accordion title="Reporting Issues">
    If you experience or witness unacceptable behavior:

    1. Contact a community organizer immediately
    2. Email contact@fyralabs.com with details
    3. Reports will be handled promptly and confidentially

    The community organizers are committed to maintaining a safe environment.
  </Accordion>
</AccordionGroup>

<Note>
  Terra follows the [Citizen Code of Conduct](https://github.com/stumpsyn/policies/blob/master/citizen_code_of_conduct.md). All community members are expected to abide by it.
</Note>

## Security and Transparency

Fyra Labs is committed to transparency and security:

### Security Updates

Security announcements are published on:
- [Twitter](https://twitter.com/TeamFyraLabs)
- [Fediverse](https://fedi.fyralabs.com/@hq)
- [Discord](https://discord.gg/5fdPuxTg5Q)

### Reporting Security Vulnerabilities

<Card
  title="Security Advisories"
  icon="shield"
  href="https://github.com/terrapkg/packages/security/advisories"
  horizontal
>
  Report security vulnerabilities through GitHub's security advisory system.
</Card>

**Important guidelines:**
- Report infrastructure or packaging vulnerabilities to Terra
- Report upstream package vulnerabilities to the upstream project first
- Do not publicize vulnerabilities until the security advisory is published
- Expect a response within 24 hours

<Note>
  For more information about Terra's security policy and processes, see the [SECURITY.md](https://github.com/terrapkg/packages/blob/main/SECURITY.md) file in the repository.
</Note>
